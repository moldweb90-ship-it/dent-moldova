# Решение проблемы с MySQL 5.6

## Проблема: "Указан слишком длинный ключ"

В MySQL 5.6 есть ограничение на длину индекса в 767 байт.
При использовании utf8mb4 (4 байта на символ) максимальная длина индексированного VARCHAR поля = 191 символ (191 * 4 = 764 байта < 767).

## Что исправлено:

1. ✅ `slug` - изменен с VARCHAR(255) на VARCHAR(191) (UNIQUE индекс)
2. ✅ `slug_ru` и `slug_ro` - изменены на VARCHAR(191)
3. ✅ `seo_canonical` - изменен на VARCHAR(191)
4. ✅ Удалены FULLTEXT индексы (могут вызывать проблемы в MySQL 5.6)

## Важно:

- Все поля с индексами (UNIQUE, PRIMARY KEY, INDEX) теперь используют VARCHAR(191) максимум
- Остальные VARCHAR(255) поля без индексов остались без изменений
- FULLTEXT поиск можно добавить позже через отдельные запросы

## Повторный импорт:

1. Удалите базу данных `dental_clinics` если она уже создана
2. Создайте новую базу `dental_clinics` с кодировкой utf8mb4_unicode_ci
3. Импортируйте обновленный файл `structure.sql`

Теперь импорт должен пройти успешно! ✅
